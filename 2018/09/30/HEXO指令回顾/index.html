HEXO指令回顾

发表于 2018-09-30 | 分类于 HEXO

本文主要参考HEXO官方文档进行撰写，记录自己的建站过程，以供自己或他人在建站的作为简单参考。
HEXO官网地址: https://hexo.io/

回顾

截止到本章节为止，已经使用了很多hexo指令了分别有：

  • init
  • generate
  • server
  • clean
  • new
  • deploy

这些指令基本上是hexo中比较常用的指令，本章会逐一介绍这些指令。

init指令

用于新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

generate指令

也可简写为

用于生成静态文件。静态文件存放在public文件夹下。

选项 描述
-d,–deploy 文件生成后立即部署网站
-w, –watch 监视文件变动

server指令

也可简写为

用于启动服务器。默认情况下，访问网址为：http://localhost:4000/。

选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

clean指令

该指令没有简写。
用于清除缓存文件 (db.json) 和已生成的静态文件 (public)。
在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

new指令

用于新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

deploy指令

也可简写为

用于部署网站。

参数 描述
-g, –generate 部署之前预先生成静态文件

其他

hexo 还有很多其他的指令，例如:

  • publish
  • render
  • migrate
  • list
  • .
  • .
  • .

这里就不一一列举了，有兴趣的话可以自行查看官方文档:https://hexo.io/zh-cn/docs/commands。
